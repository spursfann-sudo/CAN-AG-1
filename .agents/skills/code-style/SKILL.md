# Code Style Guidelines

When working in this repository, follow these rules:

- **Formatting**: We use Prettier. Do not manually format code. Always run formatting scripts if provided.
- **Linting**: All code must pass linting without warnings. Fix warnings proactively.
- **Frameworks**: Prefer Vanilla HTML/CSS/JS for simple projects unless Vite or Next.js is explicitly requested.
- **Styling**: Always use Vanilla CSS. Do not use TailwindCSS unless explicitly asked. Prefer modern, vibrant colors and glassmorphism.
